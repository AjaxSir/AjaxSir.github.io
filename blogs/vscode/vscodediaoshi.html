# vscode F5快速调试前端

  • 新建.vscode文件夹
  1. 新建launch.json文件与task.json

# launch.json 内容

{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "Run npm start",
            "type": "node",
            "preLaunchTask": "npm start", // 需要在该任务完成
            "presentation": {
                "reveal": "always",
                "panel": "new"
            }
        }
    ]
}

配置对象的字段
name：配置的名称，显示在启动配置下拉菜单中。
type：调试器类型，例如 "node"、"python"、"chrome" 等，这取决于安装的扩展。
request：调试会话的类型，通常是 "launch" 或 "attach"。"launch" 表示启动一个新的程序实例进行调试，而 "attach" 表示附加到一个已经运行的程序实例。
program：要调试的程序的入口点文件，通常用于 "launch" 类型的配置。
args：传递给程序的命令行参数。
cwd：程序的当前工作目录。
preLaunchTask：在调试会话开始前要运行的任务（来自 tasks.json）。
runtimeExecutable：用于启动程序的运行时可执行文件，例如 Node.js 或 Python 解释器。
runtimeArgs：传递给运行时可执行文件的参数。
env：为调试会话设置的环境变量。
stopAtEntry：是否在程序的入口点暂停。
console：指定控制台输出应该在哪里显示，如 "internalConsole"、"integratedTerminal" 或 "externalTerminal"。
Node.js 特定的字段
port：调试器连接的端口（仅适用于需要附加到特定端口的情况）。
skipFiles：在调试过程中要跳过的文件或文件夹，常用于忽略库代码。

# task.json 内容

{
    "version": "2.0.0",
    "tasks": [
        {
            "label": "npm start",
            "type": "npm",
			"script": "start",
            "problemMatcher": [],
            "group": {
                "kind": "build",
                "isDefault": true
            },
			"dependsOn": [ // 依赖的任务
				"build:ts"
			],
        },
		{
			"label": "build:ts",
			"type": "npm",
			"script": "build:ts",
			"isBackground": true,
			"group": {
				"kind": "build"
			},
			"problemMatcher": "$ts-webpack-watch"
		}
    ]
}

label：

类型：字符串 作用：任务的名称，用于在命令面板中识别任务。
type： 类型：字符串 作用：任务的类型，常用的类型有 "shell"（用于执行 shell 命令）和 "process"（用于直接运行可执行文件）。
command： 类型：字符串 作用：要执行的命令或脚本。
args： 类型：字符串数组 作用：传递给 command 的参数列表。
group： 类型：对象或字符串 作用：将任务分组（如 "build" 或 "test" 组），可用于快速访问。如果设置 "isDefault": true，这个任务将成为其组的默认任务。
dependsOn： 类型：字符串数组 作用：指定当前任务依赖的其他任务，这些任务会在当前任务运行前自动执行。
dependsOrder： 类型：字符串 作用：定义 dependsOn 中任务的执行顺序，可以是 "sequence"（按顺序执行）或 "parallel"（并行执行）。
problemMatcher： 类型：数组或字符串 作用：用于匹配任务输出中的错误和警告信息，以便在 VSCode 的“问题”面板中显示。
presentation： 类型：对象 作用：控制任务执行时输出的显示方式，如面板位置、是否自动聚焦等。
detail： 类型：字符串 作用：提供任务的额外描述，显示在任务选项旁边。
options： 类型：对象 作用：设置任务执行的环境变量、工作目录和 shell 配置。
